# 📦 Prompt برای درخواست Migration Package

این prompt رو کپی کن و هر وقت خواستی migration بگیری، برای من بفرست.

---

## 🎯 Prompt (کپی کن و بفرست)

```
می‌خوام یک بسته کامل migration از این پروژه بگیرم تا بتونم به اکانت دیگه منتقلش کنم.

**نیاز دارم:**

1. **فایل ZIP از تمام کدها** (بدون node_modules، .git، logs)
2. **Backup کامل دیتابیس** (JSON format)
3. **Template تنظیمات** (با placeholders برای API keys)
4. **Prompt آماده** برای اکانت جدید که کپی‌پیست کنم

**مهم:**
- فقط فایل‌هایی که لازمه رو export کن
- API keys رو به صورت placeholder بذار (امنیت)
- یک prompt ساده و گام‌به‌گام بساز که در اکانت جدید کپی‌پیست کنم
- اسکریپت‌های import/export رو هم اضافه کن

**خروجی مورد انتظار:**
1. project-code.zip
2. database-backup-*.json
3. settings-template.json
4. IMPORT_PROMPT_FINAL.md (راهنمای کپی‌پیست)

لطفاً تمام فایل‌ها رو آماده کن و بهم بده.
```

---

## 📋 توضیحات

### چرا این prompt؟

این prompt به من می‌گه که:

1. **چی می‌خوای:** یک بسته کامل برای انتقال
2. **چطور می‌خوای:** فایل‌های جداگانه، نه checkpoint
3. **چه چیزهایی لازمه:** ZIP، دیتابیس، تنظیمات، prompt
4. **چه چیزهایی نمی‌خوای:** node_modules، .git، API keys واقعی

### چی می‌گیری؟

بعد از فرستادن این prompt، من این فایل‌ها رو برات آماده می‌کنم:

| فایل | حجم تقریبی | توضیحات |
|------|-----------|---------|
| `project-code.zip` | ~400 KB | تمام کدها |
| `database-backup-*.json` | ~300 KB | دیتابیس کامل |
| `settings-template.json` | ~2 KB | تنظیمات با placeholders |
| `IMPORT_PROMPT_FINAL.md` | ~5 KB | راهنمای کپی‌پیست |

### چطور استفاده کنی؟

1. این prompt رو کپی کن
2. در چت من بفرست
3. منتظر بمون تا فایل‌ها رو آماده کنم
4. فایل‌ها رو دانلود کن
5. `IMPORT_PROMPT_FINAL.md` رو باز کن و دستورات رو دنبال کن

```
می‌خوام migration package بگیرم. 
فایل‌های ZIP، دیتابیس، و prompt آماده رو بهم بده.
```

### نسخه کامل (با جزئیات بیشتر)

```
می‌خوام یک بسته کامل migration از این پروژه بگیرم.

**شامل:**
- ZIP تمام کدها (بدون node_modules)
- Backup JSON دیتابیس
- Template تنظیمات (API keys = placeholder)
- Prompt کپی‌پیست برای اکانت جدید

```

```
migration package بده
```

## 🚀 نکات مهم

### ۱. زمان مناسب برای Migration

بهترین زمان برای گرفتن migration:

- ✅ بعد از اتمام یک feature بزرگ
- ✅ قبل از شروع تغییرات مهم
- ✅ قبل از deploy
- ✅ قبل از تعطیلات (backup)

### ۲. چیزهایی که Export نمیشن

این چیزها export نمیشن (باید دستی تنظیم کنی):

- Environment variables سیستمی (DATABASE_URL، JWT_SECRET، ...)
- API keys واقعی (به صورت placeholder export میشن)
- node_modules (باید در اکانت جدید `pnpm install` کنی)

### ۳. امنیت

- API keys به صورت placeholder export میشن
- فایل backup رو بعد از import حذف کن
- فایل settings-template.json رو بعد از استفاده حذف کن
